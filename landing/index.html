Source → Targets · Filters built-in

Migrate MySQL users with
filters, dry-runs, and reports.

mysql-user-migrate is a Go CLI that copies username/auth/host/privileges from a single source into one or many targets. It ships with include/exclude wildcards, concurrent targets, dry-run planning, JSON reports, and overwrite controls.

View on GitHub Run the CLI 
$ go run ./cmd/mysql-user-migrate \
  --source "user:pass@tcp(src:3306)/" \
  --target "name=stg=user:pass@tcp(stg:3306)/" \
  --include app_* \
  --dry-run --report report.json
✔ plan ready: 1 create · 0 updates · 0 drops
→ remove --dry-run to apply

Everything you need to migrate users

Filtering

Include / exclude

Use --include/--exclude with wildcards like app@10.0.% so only the right users move.

One-to-many

Multiple targets

Repeat --target or define them in config; --concurrency migrates several destinations in parallel.

Modes

Dry-run & reports

--dry-run plans without changes, while --report emits JSON for CI pipelines and audits.

Safety

Overwrite controls

--drop-missing and --force-overwrite keep targets clean; secrets stay masked in logs.

Opinionated workflow

  • 1 · Connect

    Pass --source and repeatable --target flags or load them via config.example.yaml.

  • 2 · Filter

    Use --include/--exclude with usernames or user@host patterns to scope the run.

  • 3 · Plan

    Run with --dry-run to view the summary, optional JSON report, and validate overwrite settings.

  • 4 · Apply

    Drop the dry-run flag, optionally enable --drop-missing or --force-overwrite, and watch the logs.

Operational notes

make deps keeps Go modules tidy, make fmt formats code, and make run ARGS="..." wraps the CLI. Secrets stay redacted in logs, and root/system accounts stay untouched unless explicitly included.

Ready-to-run CLI

Configure via environment variables or config.example.yaml. The CLI keeps secrets off logs, honors context.Context, and plays well with CI/CD.

# .env
SOURCE_DB_URI=mysql://readonly:***@dev-db:3306

# Run

  --target $TARGET_DB_URI \
  --apply"

Built for teams

Single Go binary, explicit flags, and config files so every migration is reviewable.

  • Config-first

    Ship source, targets, and filters in YAML/JSON just like config.example.yaml.

  • Concurrent targets

Command Palette

Common recipes

Dry-run a subset
  make run ARGS="--include app_* --dry-run"

Target multiple envs
  make run ARGS="--target name=stg=dsn \
                 --target name=prod=dsn \
                 --concurrency 2"

Produce a JSON report
  make run ARGS="--dry-run --report report.json"
